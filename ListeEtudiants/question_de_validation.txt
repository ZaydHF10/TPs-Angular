Questions ouvertes

Le fichier app.component.ts définit le composant racine de l’application. Il contient la classe du composant, son template, son style et son sélecteur. Il est déclaré dans app.module.ts dans le tableau declarations, puis Angular démarre ce composant via main.ts qui lance la fonction bootstrapModule(AppModule).

Pour créer un composant, on exécute la commande ng generate component nom. Angular crée quatre fichiers : .ts, .html, .css, .spec.ts. Le composant doit ensuite être déclaré dans un module, puis utilisé via son sélecteur dans un template. Cette intégration permet à Angular de reconnaître et compiler le composant.

FormsModule fournit les directives de formulaires template-driven, dont ngModel. Il contient les outils nécessaires pour gérer la saisie utilisateur, la validation et la synchronisation entre les données du composant et la vue. Sans ce module, ngModel n’est pas reconnu.

La liaison de données bidirectionnelle permet de synchroniser automatiquement la valeur du modèle et celle du champ de formulaire. Exemple :
<input [(ngModel)]="nom">
Lorsque l’utilisateur modifie la valeur, la variable nom change, et si la variable change dans le composant, la vue se met à jour.

Si un composant n’est pas déclaré dans un module, Angular ne le reconnaît pas et génère une erreur indiquant que l’élément n’est pas connu. Le composant n’est pas compilé, il n’apparaît pas dans la vue et bloque le fonctionnement de l’application.
